# Mutations

## `dryRun`

A mutation that spins up a new temporary node from the current state and emulates a given transaction. It returns a <a href="/docs/reference/objects/#receipt">`[Receipt!]!`</a> for the emulated transaction. You can optionally use UTXO validation.

**args:**

`tx`: <a href="/docs/reference/scalars/#hexstring">`HexString!`</a>

The transaction hex string.

`utxoValidation: Boolean`:
Whether or not to use UTXO validation.

## `produceBlocks`

A mutation that produces blocks, that can be used for testing that requires block advancement. Returns a <a href="/docs/reference/scalars/#u64">`U64!`</a>.

**args**

`blocksToProduce`: <a href="/docs/reference/scalars/#u64">`U64!`</a>

The number of blocks to produce.

## `submit`

A mutation that submits transaction to the transaction pool and returns a <a href="/docs/reference/objects/#transaction">`Transaction!`</a>.

**args**

`tx:` <a href="/docs/reference/scalars/#hexstring">`HexString!`</a>

The transaction hex string.
